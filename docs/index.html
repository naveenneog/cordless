A CLI-first remote terminal. Run cordless, scan the QR, drive your dev-box sessions from your phone.
Download CLI GitHub Android APK

What it is

cordless is a single self-contained command you install on your dev box or laptop. Run cordless and it opens a full-screen terminal dashboard whose starting screen shows a pairing QR — scan it with the phone app and your sessions (a shell, claude, codex, copilot, or your own custom launchers) are in your pocket. Organise them into tab groups, rename tabs, and keep going after a reboot — sessions are restored with their scrollback history. The daemon stays up after you close the dashboard, and reconnecting replays where you left off.

New in v0.8

🗂️ Tab groups

Chrome-mobile-style groups for when you're juggling >10 sessions. Collapsible group headers in the dashboard, group chips on the phone, and a smart-view filter (All · Attention · Claude · Codex · Copilot · Shell).

🚀 Custom launchers

Define your own profiles in ~/.cordless/config.json — any command, args, cwd, env. Missing executables show as unavailable instead of crashing. cordless profiles lists them all.

🤖 GitHub Copilot CLI

A built-in copilot profile alongside claude and codex, with the same waiting/finished attention detection agents get.

💾 Persisted history

A capped, plain-text scrollback is saved per session, so a reopened session after a reboot shows what it was doing before — cordless history to inspect or clear it.

✏️ Rename tabs

Rename any session — cordless rename <id> <title>, the dashboard e key, or long-press on the phone. Changes broadcast live to every connected client.

📦 Real installer

cordless setup installs the binary to a stable path, adds it to PATH, and registers autostart — no more hand-extracting a zip.

CLI-first

One self-contained binary (its own Node runtime + node-pty, no install prereq). Run cordless for a full-screen dashboard: status, a live pairing QR, and your sessions.

Attention state

Juggling many Claude / Codex sessions? cordless infers which is working, idle, or waiting for you from its output, badges and sorts them attention-first, and can ping your phone (ntfy / webhook).

Persistent sessions

PTYs survive disconnects, network switches, and app backgrounding. Reconnect replays from your last-seen byte, or a full-screen snapshot.

Attach anywhere

cordless attach <id> streams a session into your terminal; the phone app gives you the same sessions with a touch key bar and QR scanner.

Reach it anywhere

Tailscale is the recommended path; same-Wi-Fi LAN also works. No ports exposed to the internet.

Secure by default

Per-device tokens, single-use pairing codes only the local daemon can mint (loopback-only), Origin allowlist, CSP, and a daemon that only ever runs as you.

Requirements

Dev box (runs cordless)

Nothing. The cordless CLI is a self-contained binary (its own Node runtime + node-pty) for Windows or Linux. Optional: coding agents on PATH (claude, codex, copilot) for those built-in profiles, or add your own launchers. (Building from source needs Node 22+ and a C/C++ toolchain for node-pty.)

Phone

Any modern browser (install the PWA), or Android 8.0+ for the APK. Nothing else to install.

Remote access (optional)

Tailscale on the dev box and phone for secure access from anywhere. Same-Wi-Fi LAN needs nothing extra.

Only if you build the APK yourself

Node ≥ 22, JDK 21, and the Android SDK (platform 34+, build-tools 34+). Most people just grab the prebuilt APK from Releases.

Zero dependencies to install — the CLI binary bundles the Node runtime and node-pty. Under the hood the daemon uses node-pty, ws, zod, @xterm/headless, @xterm/addon-serialize, qrcode-terminal; the phone app uses @xterm/xterm, react/vite, and @capacitor/* — all pulled by npm run setup if you build from source.

Install

1 · Install cordless on your dev box / laptop

Download the self-contained cordless CLI from Releases and unpack it — it bundles its own Node runtime and node-pty, so there's nothing to install first. 

Windows   cordless-cli-windows-x64.zip    →  run cordless.exe
Linux     cordless-cli-linux-x64.tar.gz   →  run ./cordless
macOS     use the desktop app, or build from source (CLI binary coming)

Put it on your PATH, then just run it — or let cordless install itself:

cordless setup      # installs cordless to a stable path, adds it to PATH + autostart
cordless            # opens the dashboard: daemon status + a pairing QR + your sessions
cordless install    # (alternative) just register the daemon to run at login

2 · Pair your phone

The dashboard's starting screen shows a QR. Install the Android app, tap Scan QR, and scan it. (Or press p for a fresh code, run cordless pair, or open the printed URL in your phone's browser as a PWA.)

3 · From anywhere

Put the dev box and phone on the same Tailscale tailnet and lock port 7443 to your own devices with a tailnet ACL. Then the pairing QR uses a stable 100.x / *.ts.net address. Never expose 7443 to the public internet.

Manage it: cordless status · cordless doctor · cordless sessions · cordless attach <id> · cordless resume · cordless profiles · cordless group · cordless rename <id> <title> · cordless history · cordless notify test · cordless stop.

Prefer to run from source? (Node 22+) 
git clone https://github.com/naveenneog/cordless
cd cordless
npm run setup      # installs agent + client deps (builds node-pty)
npm run build      # builds the web client the daemon serves
npm start          # starts the daemon on :7443  —  then run `node agent/src/index.js` for the dashboard

npm --prefix agent run sea   # build a self-contained cordless binary into agent/dist-sea/

Desktop app (optional)

The CLI is the primary way to run cordless. If you'd rather have a window, a hardened Electron shell opens the same UI on your laptop or dev box, talking to your local daemon. Same sessions, tabs, and replay as the phone — with a real keyboard.

QR-first auth

Pairing stays QR/code based. The desktop adds one extra 🖥️ Connect to this computer button backed by a loopback-only credential.

Loopback only

That credential lives in ~/.cordless/desktop-credential.json (mode 0600) and is rejected for any non-loopback address — even a Tailscale IP. Never a remote bypass.

Not running? No blank window

If the daemon is down, the app shows a Start daemon / Retry screen instead of failing silently.

Grab an installer from Releases (Windows .exe, macOS .dmg, Linux .AppImage/.deb), or build it yourself: 

cd desktop
npm install
npm start          # run against your local daemon
npm run dist       # package installers into desktop/release/

On your phone

Security: a paired device has the same shell access as your user account — treat its token like an SSH key. cordless uses per-device hashed tokens, single-use rate-limited pairing, an Origin allowlist, and strict security headers, and never runs as root. The desktop app's local shortcut uses a loopback-scoped token that is rejected for any non-loopback address. Keep port 7443 off the public internet; use Tailscale + an ACL.
